ZMQ. Паттерн PUB-SUB. Сервер посылает клиентам сообщение "Сработало!", и они оба его одновременно печатают. Обмен происходит по порту 8005.

UDP. Клиент принимает вводимое пользователем сообщение и посылает его серверу, который принимает его и посылает обратно, 
попутно выводя ip-адрес клиента, и клиент печатает принятое сообщение. Обмен происходит по порту 8888.

TCP. Клиент посылает серверу заранее заданное сообщение, и тот выводит на экран количество байт в сообщении. Обмен происходит по порту 8888.
